The story of the project

Which kind of model actually recovers which kind of fraud?

A fraud-detection project that turned into something more interesting: instead of chasing a leaderboard number, it plants fraud with a known answer key and then measures, signature by signature, whether a tabular model, a graph network, or a sequence model is the representation that truly recovers it — and what it costs to learn structure rather than hand-craft it.

How to read this page

This is the narrated tour — the kind you'd give someone over coffee. Each section poses a question, runs an experiment, and reports a finding (including the findings that came out negative — those are often the most honest part). Every table and chart is the project's real output; jargon like GNN, SSM, and Mamba is unpacked in callouts and in the glossary.

01The problem that forced a pivot

It started as an ordinary card-fraud classifier on a public dataset (Sparkov). It almost immediately hit a wall — and the wall is the whole reason the project is interesting.

The plan was to add fancy neural methods — a graph network, a sequence model — on top of a standard classifier and show they help. But before adding them, a basic sanity check on the downloaded fraud labels revealed there was nothing structural for those methods to recover:

  • The geographic label was essentially uncorrelated with fraud (lift ≈ 1.0) — geography carried no signal.
  • The category label was saturated — it fired everywhere, so it discriminated nothing.
  • There was no ring structure, no sequence structure — nothing for a graph or a sequence model to latch onto.
The silent-failure trap

Here's the danger: if you bolt a graph network onto data with no graph structure, it doesn't error out. It "works" — it produces a number — by quietly overfitting noise. You'd write it up as a success and never know the result was meaningless. The project's first real contribution was refusing to walk into that trap.

02The pivot: don't detect the fraud — plant it

The question

If real downloaded fraud has no recoverable structure, can we instead build fraud whose structure we control exactly — so that "did the graph model recover the ring?" becomes a question with a ground-truth answer?

The thesis that replaced plain detection:

Take the legitimate transactions as background, inject fraud with known typology signatures (a ground-truth answer key), then measure which representation recovers which typology — and how detectability degrades as typologies overlap.

This flips the project from a leaderboard exercise into a controlled scientific experiment. Every neural method is now handed data designed to reward it. So each result becomes a falsifiable claim — "a graph recovers a ring; plain per-row features cannot" — instead of an unexplained number.

The name "cross-border" is a pun that captures the design: it's not about geography. A transaction is "cross-border" when it crosses the behavioral boundaries of several fraud types at once (it's multi-label). Those overlap events are how the project studies the hardest case — when signatures pile up on a single transaction.

03Five typologies, each mapped to the model that should catch it

The heart of the design is a deliberate pairing: each planted fraud pattern is matched to the representation whose inductive bias makes it the natural detector. The experiment is whether that pairing actually holds.

ring

K distinct cards all hit one merchant inside a short time window (a coordinated fan-in).

→ Graph network

velocity

One card, many transactions packed into a short burst.

→ Sequence / tabular

temporal

A transaction at one of this card's rarest hours-of-day (a per-card anomaly).

→ Sequence model

category

A transaction in one of the card's rarest spending categories.

→ Tabular model

geo

A merchant placed implausibly far from the card's home location.

→ Tabular model

Why these pairings

A ring is invisible in any single transaction row — it only exists in the relationships between cards and merchants, which is exactly what a graph encodes. A temporal anomaly is "rare for this card," which needs the card's whole history — a per-card sequence. Geo and category, by contrast, are visible in a single row (a distance, a category flag), so a plain tabular model should already nail them. The experiment tests whether reality matches this intuition.

04The detail that makes the answer key trustworthy: leak-free injection

An answer key is only as good as the care taken to avoid accidental giveaways. If injected fraud differed from legit traffic in some unintended way — say, slightly larger amounts — a model could "detect the ring" by reading that artifact instead of the real ring structure. The whole experiment would be a lie.

The controlled-benchmark invariant

An injected row matches the legit distribution on every axis except the one intended signature.

Concretely: amount and timestamp are sampled from the real legit pools; merchant coordinates reuse the legit home→merchant offset distribution (so injected fraud sits as close to home as ordinary traffic — except for geo); temporal injections keep a real legit date and perturb only the hour-of-day. So a detector can only win through the genuine planted signal, never a distributional shortcut. The answer key lives in two columns (typology, inj_event), and a pytest suite enforces the no-leak invariant on every build.

05The bake-off: which representation wins each slot?

The question

For each planted signature, can a tabular model, a graph model, or a sequence model recover it — and how close does a learned model get to a hand-crafted "oracle" feature that's simply handed the answer?

Every score is isolated solo-vs-legit AUC: each signature's pure (non-overlapping) rows scored against legit only. This deliberately stops, say, a geo+ring overlap from letting the ring "borrow" geo's easy distance signal. (AUC 0.5 = coin flip; 1.0 = perfect.)

The finding

For a ring, a per-row tabular model is at chance (0.58) — it literally cannot see merchant-side structure. The graph network recovers most of the signal (0.84). For a temporal anomaly, global clock features top out at 0.70 because "3am" isn't anomalous in general — only for a card that never shops at 3am; the sequence model gets to 0.81. In both, the learned model recovers most of the signal end-to-end, and the gap to the oracle is the honest price of learning structure rather than being handed it. The other three slots are genuinely tabular — itself a result, not a failure.

What "GNN" and "SSM" mean here

GNN (graph neural network, here a GraphSAGE called RingSAGE): builds a graph of cards and merchants and lets each node "listen" to its neighbours. A ring lights up because one merchant suddenly has many distinct card-neighbours in a short window. Crucially the model is not handed the count — it must learn fan-in by message-passing.
SSM (state-space model, the S4 → Mamba family): a sequence model that summarises a card's history into a running state. Here it keeps decaying "memory" of which hours a card usually transacts at, so a transaction at a personally-rare hour stands out. A clever trick — fixing the memory decay instead of learning it — lets the entire per-card state be precomputed with a signal-processing filter, cutting the run from ~51 min to ~2.5 min (≈20×) with no backprop-through-time.

06The production architecture: neural nets propose, the GLM decides

A guiding principle keeps the project disciplined: the neural networks are feature extractors, not classifiers. They each distil their representation into a few scalar numbers; a plain, interpretable logistic-regression GLM remains the actual decision-maker.

raw data ─► [GNN] ─► structural scalars ┐ raw data ─► [SSM] ─► temporal scalars ├─► X ─► binary-relevance GLM ─► 5 labels classical features ───────────────────────┘ (each neural scalar must first pass an LR-test gate)
The "LR-test gate" in plain terms

Before a neural feature is allowed into the classifier for a given label, it must pass a likelihood-ratio test — a formal statistical check that it adds real explanatory power over the tabular baseline. If it doesn't help, it's dropped. This keeps the system honest: no neural feature rides along unless it earns its slot.

This design buys three things at once: interpretability (the final model is a logit you can read), statistical rigour (every extension is gated by a hypothesis test), and a clean way to attribute credit (a feature that helps rings but not geo is admitted only for rings).

07Does it actually work end-to-end? The nested test

The question

The bake-off tested each representation in isolation. But the product is the five-label classifier. If we wire the admitted neural features into it and rebuild, does the multi-label model improve — and does it improve through precisely the labels each representation was built for?

08Going deeper: does fancier architecture close the gap? (The Mamba question)

Both learned models sit below their oracle (ring 0.84 vs 0.96; temporal 0.81 vs 0.88). The natural next question — most relevant to a state-space-modelling portfolio — is whether a more expressive architecture closes that gap. Two probes were run; both came back with clean, instructive answers.

Probe A — a "selective" (true-Mamba) sequence model for temporal

Mamba's signature idea is selectivity: the model learns, per input, how much to remember — input-dependent memory. The fixed temporal SSM uses a frozen memory bank. So a learned-selective version should win... right?

Temporal extractorisolated AUC

The finding (a clean negative)

The selective model scored 0.780 — below the fixed model's 0.806. Selectivity didn't help; it regressed. And the reason is principled: a card's hour-of-day distribution is stationary — it doesn't change based on the current transaction — so there is nothing for input-dependent memory to exploit. Selectivity pays off when what to remember depends on content; here it just spends capacity it can't recover. (It's still LR-admitted, G²=2,417 — it carries real signal, just less than the fixed bank.)

Probe B — a snapshot-graph GNN for ring

Maybe the ring gap comes from a crude time-bucketing artifact (a ring straddling a bucket boundary gets split). A "snapshot" graph with overlapping centred windows fixes that exactly.

Result: just +0.008. The boundary artifact was real but minor — ~94% of the gap is the inherent cost of recovering an exact distinct-card count by message-passing rather than reading it off directly.

The bracket — the project's signature scientific insight

Put the two probes together and you get a mechanistic map of the Mamba inductive bias: input-dependent state-space machinery helps when what-to-remember is content-dependent (velocity's irregular timing) and is dominated by a simple fixed memory bank when the target is stationary (temporal's hour distribution). Neither architectural lever buys back the oracle gap — that gap is just the price of learning structure end-to-end. A far more useful finding than "model X scored higher."

09The state-space showcase: the whole S4 → Mamba lineage, from scratch

To turn that one-off bracket into a real demonstration, the project implements the entire modern sequence-model family from scratch and races them head-to-head on the same two slots — plus RNN, TCN, and Transformer baselines.

The engineering centrepiece

All the diagonal state-space models (LRU, S5, DSS, Mamba-S6) reduce to one primitive: a parallel associative scan — a work-efficient, log-depth way to compute a sequential recurrence in parallel (so it trains in O(T log T) without a slow Python loop). Written from first principles, handles real and complex states, unit-tested against a sequential reference. Each model in the zoo differs only in how it parameterises one term.

Ablations — the design choices, as evidence rather than folklore

The finding

Across the whole lineage, input-dependent / state-space machinery helps when what-to-remember is content-dependent (velocity), and is dominated by a fixed low-dim bank when the target is stationary (temporal). §08's two one-off probes were not a fluke — the pattern is architecture-wide. And the elaborate levers (HiPPO init, learned selectivity, large state) only pay off where the signal actually demands them.

11The reality check: does any of this transfer to real fraud?

The benchmark is built on a simulator. So the obvious challenge: does it say anything about real fraud? Tested on the fully-anonymised real-world IEEE-CIS dataset (590k transactions). The injection itself can't transfer (real data has no merchant id, no coordinates, no clean card id) — but two things can be checked.

The finding — the premise is confirmed

Real fraud is not a single clean typology — the oracles that ace planted data are at chance on real fraud, while a broad tabular model carries it at 0.83. This validates the whole approach: because real fraud doesn't arrive pre-labelled by mechanism, you have to plant signatures to measure which representation recovers what. The fold bounds the claims rather than overselling them.

12From a study to a reusable benchmark

The final move reframes the project for a real venue (the NeurIPS Datasets & Benchmarks track): not "a study on one dataset," but a reusable protocol — plant typed signatures with a known answer key into any transaction-stream dataset, then measure representation recovery.

This required decoupling the injection code from Sparkov's specific columns via an abstract Schema (a role→column map), then writing thin adapters for two more public datasets (PaySim, BankSim). A hard invariant governed the refactor: the original Sparkov build had to stay byte-identical (verified at exact equality) so no documented number could silently shift.

The finding — a dataset-capability result, not a bug

The identical code recovers rings on Sparkov and on a synthetic unit-test frame, so this isn't a code failure — it's a property of the datasets. The ring (graph) slot demands both continuous timestamps and persistent multi-transaction entities; sequence slots tolerate coarser data; tabular slots need only the relevant column. This becomes a "capability matrix" that tells any future user exactly which dataset properties each representation slot requires — precisely the kind of contribution a benchmark paper is made of.

13Honest limitations (stated plainly — they defend the work)

  • The base data is synthetic. Sparkov, PaySim, and BankSim are simulators; their fraud is cleaner than reality (geo separating perfectly is a symptom). Acknowledged and partly the point — the contribution is the controlled protocol, the only way to measure recovery at all.
  • The recovery thesis can't be tested on real fraud — that needs a per-typology answer key, which only injection provides. The IEEE-CIS fold validates the premise and bounds external relevance; it doesn't extend the recovery claim.
  • Cross-dataset replication is partial. Velocity replicates on all three bases; temporal and ring are data-blocked on the alt bases (documented, with mechanism, in the capability matrix).

Throughout, results are recorded as-found — the negatives (selectivity regressing temporal, category headroom un-manufacturable, the large-n p-value inflation) are reported as prominently as the wins. That discipline is the project's quiet thesis.

14What the project demonstrates

  1. A reframe that creates rigour. Turning "detect fraud" into "plant fraud with an answer key, then measure recovery" converts vague claims into falsifiable experiments — and dodges the silent-failure trap that bites naive neural add-ons.
  2. A mechanistic map of the Mamba inductive bias. Selectivity helps content-dependent timing (velocity) and hurts stationary distributions (temporal) — demonstrated across the entire S4→Mamba lineage, not asserted.
  3. Deep, from-scratch state-space modelling. A parallel associative scan unifying LRU/S5/DSS/Mamba-S6, a fixed-vs-learned speed trick (~20×), full ablations — the resume-grade core.
  4. Statistical discipline. Neural features gated by likelihood-ratio tests, a GLM kept as the interpretable classifier, and a repeated insistence on effect size over p-values at large n.
  5. Honesty as a feature. Negative and null results (category, geo, selective SSM, snapshot GNN, external transfer) are first-class outcomes — each named with its mechanism.
  6. A path to a real venue. Generalised to a base-agnostic protocol with a capability matrix across three datasets, a full test suite (57 passing), and a one-command reproducible pipeline.
The 30-second version

"I built a controlled benchmark that plants five kinds of fraud with a known answer key, then measured which model architecture recovers which kind. Graph networks recover rings, sequence models recover per-card timing anomalies, and plain tabular models handle the rest. Going deeper, I mapped exactly where the Mamba 'selectivity' mechanism helps and where a simple fixed memory bank beats it — implementing the whole S4→Mamba family from scratch to show it. Every neural feature has to pass a statistical gate before it's used, and the negative results are reported as carefully as the wins."
